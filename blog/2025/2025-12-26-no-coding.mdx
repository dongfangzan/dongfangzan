### 那个改变一切的夜晚
2025年12月26日，一个普通的冬天 ❄️

我坐在电脑前，像往常一样准备开始写代码。

但这一次，我没有打开 IDE，没有新建文件，没有敲下第一行 `import`。

我只是打开了一个软件

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261612251.png)

然后，我开始"说话"：

> "帮我写一个通用后台管理系统，就像 RuoYI 一样，支持用户管理、角色管理、菜单管理、操作日志管理……"

**几分钟后**，一个完整的系统，出现了在我的面前。

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261615403.png)

那一刻，我意识到：**我不再需要"写"代码了。**

{/* truncate */}

### 十年：写代码的方式从未改变

2014年，我硕士毕业 🎓

这十年，我走过不少地方：国企、互联网公司、事业单位，后来又做了几年数字游民，接单写代码。

环境在变，节奏在变，但我的写代码方式始终没变：

```
需求来了 → 打开 Google → 搜索 "Java 实现 xxx" 
→ 翻十几页结果 → 复制粘贴 → 改改改 → 跑通
```

**搜索引擎是我最好的朋友，Stack Overflow/CSDN/博客园 是我的第二个家。**

### ChatGPT 带来的变革

2023年，ChatGPT 横空出世 🔥

我也开始用 AI 辅助写代码，但那时候 AI 对我来说，只是一个更聪明的搜索引擎：

- 它给我答案
- 但代码还是要我自己改
- Bug 还是要我自己调

**但我依然是"写代码的人"，AI 只是帮我写得快了一点。**

2025年，国产 AI 模型开始崛起 🇨🇳

DeepSeek、智谱、月之暗面... 一个个名字刷屏。

最直接的感受：

| 以前 | 现在 |
|:---|:---|
| 🌍 需要翻墙 | ✅ 国内直连 |
| ⏳ 等待 loading | ✅ 秒级响应 |
| 💸 价格昂贵 | ✅ 便宜亲民 |
| 🤔 中文一般 | ✅ 原生优化 |

**选择更多了，体验更好了，门槛更低了。**

## 真正的 Vibe Coding
到了 2025 年 3 月，Claude Code 出现了 🤖

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261648560.png)

号称是 Vibe Coding 的标杆，大家都说好用。

但我试了之后，真的是特别强，特别好用

但也还是那两个老问题：

| 问题 | 体验 |
|:---|:---|
| 🐌 **卡** | 响应慢，经常转圈 |
| 💰 **贵** | 用不起，钱包受不了 |

尤其是命令行编程的模式，跟我用了十年的"手写代码"习惯完全冲突。

打开 IDE、敲代码、改代码、跑代码，这套流程已经刻进肌肉记忆了。

**突然让我对着命令行说话，说实话，不愿意尝试。**

## 终于找到「对味儿」的 Vibe Coding 🎯
在我以为接下来的几年之内，我可能还是会继续用人写 + AI 辅助的方式来写大型项目。

也会尝试用 Vibe Coding 来写一些简单的脚本，或者是一些小项目。

但随着 GLM4.7 和 ZCode 的发布，好像一切都发生了一点点变化。

### 首先国产模型也比肩最强了
在大模型排行榜上，GLM4.7 超过了最新的 GPT-5.2，接近 gemini 和 Claude 等顶尖模型

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261744062.png)

### 其次是成本上的优化
购买官方的接口，3 倍的用量，1/7 的价格

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261728950.png)

### 再次是工具上的革新

Z Code 是一个编码工具，它将最优秀的 AI 智能体与你现有的工具完美结合，让你能够无缝地进行规划、编码、审查和部署。

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261756412.png)

他可以直接使用官方的 Claude 模型、智谱系列模型

还可以使用你自定义的模型，并将其直接接入到 Claude Code 中

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261758366.png)

针对于不喜欢命令行编程的用户，Z Code 提供了一个基于 GUI 的界面，你可以直接在界面上"说话"，直接看到结果

![](https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261800601.png)

彻底解决了Claude Code 的三大痛点 💥

| 痛点 | Claude Code | Z Code |
|:---|:---|:---|
| 🐌 **响应慢** | 经常卡顿，等待漫长 | ⚡ 秒级响应，丝滑流畅 |
| 💰 **成本高** | 价格昂贵，钱包压力 | 💵 便宜亲民，性价比高 |
| 🖥️ **命令行** | 黑框操作，学习门槛高 | 🎨 GUI 界面，直观易用 |

### 最后，最重要的是速度上质的飞跃
对于我来说，GLM 4.7 带给我的最大的影响，其实是速度上的飞跃。

<img src="https://hx-zsy.oss-cn-chengdu.aliyuncs.com/img/202512261717314.png" alt="alt text" width="200" />

如果你有私有化部署的 GLM4.7-FP8 在 H100 上运行，decode 的速度会达到**惊人的 200 tokens/秒**。

即使是稍差一点的 H20，decode 的速度也会达到 **160 tokens/秒**。

**这是我用过速度最快的旗舰模型，没有之一。**

别小看这点速度的提升，它对编程体验的改变是颠覆性的 💥

#### 为什么速度这么重要？🤔
**想象一下这样的场景：**

| 速度 | 生成 1000 行代码 | 你的体验 |
|:---|:---|:---|
| 🐌 20 tokens/秒 | 约 50 秒 | 等待、走神、刷手机 |
| 🚀 160 tokens/秒 | 约 6 秒 | 专注、连贯、心流状态 |

**当代码以"说话"的速度生成时：**

- ✅ **思维不会被打断** — 你的思路还在，代码就已经出来了
- ✅ **反馈即时可见** — 改完需求，马上看到结果
- ✅ **沉浸式编程** — 进入心流状态，效率翻倍
- ✅ **多轮迭代更轻松** — 一分钟内可以完成 3-5 次调整

现在，我们回到一个老生常谈的问题

## 程序员会被 AI 取代吗？
我觉得会，也不会。

会被替代的，是从前写重复写业务代码的人，即使懂 Java、Python、Golang...

不会被替代的，是那些把时间花在解决问题上的人。

**Vibe Coding 🤖 的意义就在于**

**🎯 让我们从"写代码的人"变成"解决问题的人"**

- 不再纠结语法细节
- 不再重复造轮子
- 专注于业务逻辑和架构设计
- 把更多时间花在真正有价值的事情上

这就是从今天开始，**我不再"写"代码的原因。**

## 最后的话 📝

十年前，我学会了写代码。

这不是倒退，而是进步。

**因为我们终于可以专注于真正重要的事情——用技术解决问题，创造价值。**
